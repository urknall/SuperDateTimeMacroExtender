PLUGIN_SUPERDATETIMEMACROEXTENDER_HELP_JSON_TITLE
	EN	JSON endpoint format
	DE	JSON-Endpunkt-Format

PLUGIN_SUPERDATETIMEMACROEXTENDER_HELP_JSON_INTRO
	EN	Provide a JSON document with a top-level "result" array. Each element in result[] is a record whose fields can be used in macros.
	DE	Stelle ein JSON-Dokument mit einem "result"-Array auf oberster Ebene bereit. Jedes Element in result[] ist ein Datensatz, dessen Felder in Makros verwendet werden können.

PLUGIN_SUPERDATETIMEMACROEXTENDER_HELP_NOTE_MULTIURL
	EN	Multiple URLs are supported. The plugin fetches them in order and merges their result[] records. If the same id/idx/Name exists multiple times, the entire record from the later URL replaces the earlier one completely (fields are not merged).
	DE	Mehrere URLs werden unterstützt. Das Plugin holt sie der Reihe nach und fügt die result[]-Datensätze zusammen. Wenn id/idx/Name mehrfach vorkommt, ersetzt der vollständige Datensatz der späteren URL den früheren komplett (Felder werden nicht zusammengeführt).

PLUGIN_SUPERDATETIMEMACROEXTENDER_HELP_JSON_FIELDS
	EN	Lookups: use result[].id for ~e...~, result[].idx for ~i...~, and result[].Name for ~n...~. For values you can expose any fields you like (e.g. Value, Data, Unit, ValueInt).
	DE	Zuordnung: result[].id für ~e...~, result[].idx für ~i...~ und result[].Name für ~n...~. Als Werte kannst du beliebige Felder bereitstellen (z.B. Value, Data, Unit, ValueInt).

PLUGIN_SUPERDATETIMEMACROEXTENDER_HELP_JSON_RAW
	EN	Alternative formats: Instead of "result", you can use {"raw": {"1": 12.3, "2": "On"}} or provide key-value pairs at the root level {"1": 12.3, "2": "On"}. In these formats, each key becomes a record with Name="Key_<k>", id=<k>, idx=<k> (if numeric), and Value=<v>.
	DE	Alternative Formate: Statt "result" kannst du {"raw": {"1": 12.3, "2": "On"}} verwenden oder Schlüssel-Wert-Paare auf oberster Ebene {"1": 12.3, "2": "On"}. In diesen Formaten wird jeder Schlüssel zu einem Datensatz mit Name="Key_<k>", id=<k>, idx=<k> (falls numerisch) und Value=<v>.
